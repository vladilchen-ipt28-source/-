# Лабораторна робота №3: Візуалізація даних

## Опис завдання
* **Вибір датасету:** З репозиторію UCI обрано багатовимірний датасет **Automobile**, який містить категоріальні, цілі та дійсні ознаки, а також пропущені значення (позначені як `?`).
* **Посилання на опис датасету:** Детальний опис усіх ознак та характеристик можна знайти на офіційній сторінці: [UCI Machine Learning Repository — Automobile Dataset](https://archive.ics.uci.edu/dataset/10/automobile).
* **Попередня обробка (Data Cleaning):** Заміна текстових маркерів пропусків на валідні об'єкти `NaN`, видалення критичних дефектів, заповнення решти пропусків медіанними значеннями стовпців та приведення ознак до числових типів даних.
* **Аналітична візуалізація (5–8 графіків):**
  * Базові двовимірні залежності ознак (*Scatter Plot*).
  * Гістограма розподілу з примусовим кроковим розбиттям на 5 фіксованих користувацьких діапазонів та підрахунком елементів.
  * Багатовимірний аналіз складних структур (3D та 4D графіки з використанням параметрів `hue` та `size` маркерів за гайдлайнами Medium).
  * Фасетування даних (*FacetGrid*) для сегментації підграфіків за категоріальними ознаками.
  * Побудова діаграм розмаху (*Boxplot*) для аналізу викидів, теплокарти кореляційної матриці ознак (*Heatmap*) та матриці парних залежностей (*Pairplot*).

## Інструкція з налаштування середовища та запуска

### 1. Активація віртуального середовища
Відкрийте терміналі та переконайтеся, що перебуваєте в корені репозиторію. Активуйте створене раніше ізольоване середовище:

```bash
# Для Windows:
venv\Scripts\activate

# Для Linux / MacOS:
source venv/bin/activate
```
### 2. Перехід до папки лаби та встановлення залежностей
Перейдіть у папку третьої лабораторної роботи та встановіть бібліотеки візуалізації через локальний файл конфігурації:

```bash
cd lab3
pip install -r requirements.txt
```
### 3. Інструкція щодо отримання датасету
Датасет завантажується повністю автоматично під час першого запуску коду в блокноті. Скрипт перевіряє наявність локального файлу і, якщо він відсутній, самостійно звертається до API репозиторію UCI, завантажує файл imports-85.data та зберігає його як automobile.csv локально.

### 4. Запуск блокнота
Відкрийте Jupyter Notebook або скористайтеся розширенням Jupyter у VS Code для послідовного виконання комірок у файлі lab3.ipynb.
